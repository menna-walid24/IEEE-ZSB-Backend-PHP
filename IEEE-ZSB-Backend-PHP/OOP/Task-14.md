**1-Class vs object**


Class :is a blueprint that contains some functions that do different task.


class has  general properties  and each object uses these properties and change only it's value.


function inside class->method



function outside class->function



Object: is an instance class  that represent properties.



**Example:**


If a school has students and each student has his own properties like name ,id ,degree and age.


class will be students and objects will be student1 ,student2,etc.


For each student , he has these general properties but with  different values.

**3- public vs protected vs private**


public :means that any code can access it.


private :can  be accessed  only from the same class.


protected :only can be accessed in the same class or inheritance.

**4-Typed Properties:**


Is the process of defining data type like string ,int,etc .



It help prevent bugs as it allows only one type of data .



Like if only string data is needed like names it will prevent adding integer data or any other type.
